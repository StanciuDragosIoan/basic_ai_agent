v1
Create a DocumentFragment
Return a container of DOM nodes without touching the page yet
What is a DocumentFragment? A DocumentFragment is an invisible lightweight container. You build up DOM nodes inside it, then append the whole thing to the real DOM in one shot — no repaints until you're ready. Here we just put plain text inside it using createTextNode().
javascript
function formatContent(text) {
  const fragment = document.createDocumentFragment();

  fragment.appendChild(document.createTextNode(text));

  return fragment; // caller does: element.appendChild(formatContent(text))
}

v2
Inline backtick code
Detect `backtick` spans and wrap them in <code>
The key trick: split() with a capture group When you pass a regex with parentheses to split(), the matched delimiters are kept in the result array. So "hi `world`".split(/(`[^`\n]+`)/g) gives ["hi ", "`world`", ""] — letting us detect and wrap the backtick span. [^`\n] means "any char except a backtick or newline" so we never match across lines.
javascript — what's new
const parts = text.split(/(`[^`\n]+`)/g);
//                         ^ captures the match ^

parts.forEach((part) => {
  if (part.startsWith('`') && part.endsWith('`')) {
    const code = document.createElement('code');
    code.className = 'inline-code';
    code.textContent = part.slice(1, -1); // strip the backticks
    fragment.appendChild(code);
  } else {
    fragment.appendChild(document.createTextNode(part));
  }
});

v3
Triple-backtick code blocks
Detect ```fenced blocks``` and render them as <pre><code>
[\s\S]*? — matching across newlines, lazily The dot . skips newlines by default. [\s\S] means "whitespace OR non-whitespace" = literally any character including newlines. The *? is lazy — it stops at the first closing ``` instead of the last. We split the whole text on these fences, then handle each chunk.
javascript — what's new
const blocks = text.split(/(```[\s\S]*?```)/g);
//                          ^^^ lazy multiline match ^^^

blocks.forEach((block) => {
  if (block.startsWith('```')) {
    const codeText = block.slice(3, -3); // strip opening/closing ```

    const pre = document.createElement('pre');
    const codeEl = document.createElement('code');
    codeEl.textContent = codeText;
    pre.appendChild(codeEl);
    fragment.appendChild(pre);
  } else {
    fragment.appendChild(document.createTextNode(block));
  }
});

v4
Combine block and inline parsing
Two-level split: first on ``` blocks, then on `inline` spans
Why two levels? We can't run both regexes at once — a ``` block could contain backticks inside it. Solution: first split the whole text on ``` fences. For each chunk that is not a code block, run the inline backtick split on it. This gives correct nesting with no conflicts.
javascript — what's new
blocks.forEach((block) => {
  if (block.startsWith('```')) {
    // ... render as pre/code (same as v3)
  } else {
    // NEW: run inline backtick split on non-block text
    const parts = block.split(/(`[^`\n]+`)/g);
    parts.forEach((part) => {
      if (part.startsWith('`') && part.endsWith('`')) {
        const code = document.createElement('code');
        code.className = 'inline-code';
        code.textContent = part.slice(1, -1);
        fragment.appendChild(code);
      } else if (part) {
        fragment.appendChild(document.createTextNode(part));
      }
    });
  }
});

v5
Language label detection
The first line after ``` is the language tag — extract and display it
indexOf('\n') splits language from code After stripping the outer ```, the first line is the language (e.g. js). inner.indexOf('\n') finds where that first line ends. Everything before is the language tag, everything after is the actual code. If there is no newline, there is no language — just raw code.
javascript — what's new
const inner   = block.slice(3, -3);        // strip ```
const newline = inner.indexOf('\n');       // find first newline
const lang     = newline > -1
  ? inner.slice(0, newline).trim()         // text before first \n
  : '';
const codeText = newline > -1
  ? inner.slice(newline + 1)               // text after first \n
  : inner;

if (lang) {
  const label = document.createElement('span');
  label.className = 'code-lang';
  label.textContent = lang;
  wrapper.appendChild(label);
}

v6 — final
Copy button
One-click copy with visual "Copied!" feedback using a closure
navigator.clipboard.writeText() + closure The button's onclick closes over codeEl, so it always copies the correct block's text regardless of how many blocks are on the page. writeText() returns a Promise — we wait for it before showing "Copied!", then reset after 2 seconds with setTimeout.
javascript — what's new
const btn = document.createElement('button');
btn.className = 'copy-btn';
btn.textContent = 'Copy';

btn.onclick = () => {
  navigator.clipboard.writeText(codeEl.textContent).then(() => {
    btn.textContent = 'Copied!';
    btn.classList.add('copied');
    setTimeout(() => {
      btn.textContent = 'Copy';
      btn.classList.remove('copied');
    }, 2000);
  });
};

wrapper.appendChild(btn); // appended to wrapper so it layers over the pre
